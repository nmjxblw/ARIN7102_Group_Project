# 数据清洗说明

> 本文档详细说明从原始数据集到增强药物表 `enhanced_drug_table.csv` 的完整清洗、匹配与合并流程。

---

## 一、总体流程概览

```
Step 1: 加载与预处理各数据源
        ↓
Step 2: 构建药物→适应症全量映射（DS1 + DS2 + DS4）
        ↓
Step 3: 以 DS2 为底表，合并药物属性
        ↓
Step 4: 聚合同名药物（按 drug_name 去重）
        ↓
Step 5: 模糊匹配 disease_keys.json
        ↓
Step 6: 补充疾病描述（DS6）
        ↓
Step 7: 输出最终增强表
```

---

## 二、各步骤详细说明

### Step 1: 加载与预处理

**1.1 统一药名格式**

所有数据集中的药名统一为 **小写、去除首尾空格**：

```python
drug_name = drug_name.strip().lower()
```

**1.2 统一疾病/适应症名格式**

```python
condition = condition.strip()  # 保留原始大小写用于 original_conditions
condition_normalized = condition.strip().lower().replace('_', ' ')  # 用于模糊匹配
```

**1.3 处理缺失值**

- 字符串字段: `NaN` → 空字符串 `""`
- 数值字段 (`rating`, `no_of_reviews`): `NaN` → 不参与聚合计算

**1.4 HTML实体解码**

部分数据集（尤其 DS4）含有 HTML 实体（如 `&#039;`），使用 BeautifulSoup 解码：

```python
from bs4 import BeautifulSoup
text = BeautifulSoup(text, "html.parser").get_text()
```

### Step 2: 构建药物→适应症全量映射

从三个数据源收集所有 drug ↔ condition 关系：

| 数据源 | 药名字段 | 适应症字段 | 贡献 |
|--------|---------|-----------|------|
| DS1 | `drug` | `disease` | 14,685 条映射 |
| DS2 | `drug_name` | `medical_condition` | 2,932 条映射（含丰富属性） |
| DS4 | `drugName` | `condition` | ~215,000 条映射（含评分） |

**合并逻辑**:

```python
# 从各数据源提取 (drug_name, condition) 对
pairs_ds1 = ds1[['drug', 'disease']].rename(columns={'drug': 'drug_name', 'disease': 'condition'})
pairs_ds2 = ds2[['drug_name', 'medical_condition']].rename(columns={'medical_condition': 'condition'})
pairs_ds4 = ds4[['drugName', 'condition']].rename(columns={'drugName': 'drug_name'})

# 统一格式后合并去重
all_pairs = pd.concat([pairs_ds1, pairs_ds2, pairs_ds4])
all_pairs['drug_name'] = all_pairs['drug_name'].str.strip().str.lower()
all_pairs = all_pairs.drop_duplicates()
```

**按药名聚合**:

```python
# 每种药物的所有适应症 → 列表
drug_conditions = all_pairs.groupby('drug_name')['condition'].apply(
    lambda x: list(set(x.dropna()))
).reset_index()
drug_conditions.columns = ['drug_name', 'original_conditions']
```

### Step 3: 以 DS2 为底表合并药物属性

DS2 是属性最丰富的数据源，以其为基础构建属性表：

```python
# DS2 中同一药物可能对应多个 condition（多行），
```

**保留字段**: `drug_name`, `generic_name`, `drug_classes`, `brand_names`, `side_effects`, `activity`, `rx_otc`, `pregnancy_category`, `csa`, `alcohol`, `related_drugs`, `medical_condition_description`

**DS3 补充**: 对 DS2 中缺失的 `activity`, `rx_otc`, `pregnancy_category` 字段，从 DS3 中回填。

### Step 4: 评分聚合

**DS2 评分**:

```python
ds2_ratings = ds2.groupby('drug_name').agg(
    ds2_avg_rating=('rating', 'mean'),
    ds2_total_reviews=('no_of_reviews', 'sum')
)
```

**DS4 评分**（大规模用户数据）:

```python
ds4_ratings = ds4.groupby('drugName').agg(
    ds4_avg_rating=('rating', 'mean'),
    ds4_total_reviews=('rating', 'count')
)
```

**综合评分**（加权平均，按评价数量加权）:

```python
total = ds2_total_reviews + ds4_total_reviews
avg_rating = (ds2_avg_rating * ds2_total_reviews + ds4_avg_rating * ds4_total_reviews) / total
total_reviews = total
```

**层级 4: 模糊字符串匹配**

对经过前三层仍未匹配的 condition，使用编辑距离/token相似度：

```python
from difflib import SequenceMatcher

def fuzzy_match(condition, disease_keys, threshold=0.6):
    condition_normalized = condition.lower().replace('_', ' ')
    best_match = None
    best_score = 0
    for key in disease_keys:
        key_normalized = key.replace('_', ' ')
        score = SequenceMatcher(None, condition_normalized, key_normalized).ratio()
        if score > best_score:
            best_score = score
            best_match = key
    if best_score >= threshold:
        return best_match
    return None
```

**层级 5: 兜底**

所有未匹配项归入 `"others"`：

```python
if not matched_keys:
    matched_keys = ["others"]
```

#### 5.3 匹配执行顺序

```
对于每个 original_condition:
  1. 精确匹配（标准化后比较）→ 命中则加入
  2. 同义词表匹配 → 命中则加入
  3. 包含匹配（双向关键词检查）→ 命中则加入
  4. 模糊匹配（SequenceMatcher ≥ 0.6）→ 命中则加入
  5. 以上全未命中 → 该 condition 对应 "others"

最终：一个药物的所有 original_conditions 匹配结果取并集 → matched_disease_keys
```

## 三、数据质量处理

### 3.1 去重策略

- **药名去重**: 同名药物（忽略大小写）合并为一行
- **适应症去重**: 同一药物的重复适应症只保留一次
- **属性冲突**: 同名药物多行属性不一致时，取 `no_of_reviews` 最多的行

### 3.2 缺失值处理

| 字段 | 处理方式 |
|------|---------|
| `generic_name`, `drug_classes`, `brand_names` | 保留空字符串 |
| `side_effects` | 保留空字符串 |
| `rating` | 仅在有评分数据的行参与计算 |
| `activity` | DS2 缺失时从 DS3 补充 |
| `matched_disease_keys` | 至少为 `["others"]` |

### 3.3 数据验证

- 检查 `drug_name` 唯一性
- 检查 `matched_disease_keys` 中的值是否都在 disease_keys.json 中
- 检查 `avg_rating` 范围 [1, 10]
- 输出统计摘要：总药物数、匹配到disease_keys的药物比例、others占比
