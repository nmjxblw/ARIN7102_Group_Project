# 增强数据的使用 —— 未充分利用数据集的推荐方案

> 本文档说明当前增强药物表 (`enhanced_drug_table.csv`) 中 **未直接利用** 或 **仅部分利用** 的数据集，以及它们在药物推荐系统中的推荐利用方式。

---

## 一、部分利用的数据集

### DS4: kuc-hackathon-winter-2018（用户评价数据）

**当前利用**: 仅聚合了 `rating`（平均评分）和评价数量。

**未利用部分**: 约 21 万条用户评论文本（`review` 字段）。

**推荐利用方式**:

1. **情感分析 & 疗效提取**
   - 对 `review` 文本进行情感分析（正面/负面/中性）
   - 提取用户提及的具体疗效关键词（如 "worked great", "no more pain"）
   - 生成每种药物的 **真实有效率**（高评分 + 正面情感的占比）

2. **副作用真实频率统计**
   - 用 NLP 从评论中提取用户报告的副作用
   - 对比 DS2 中官方副作用列表，生成 **用户实际体验的副作用频率排行**
   - 字段建议: `user_reported_side_effects`（JSON 格式，含频率）

3. **药物对比分析**
   - 按 `condition` 分组，对同一疾病的不同药物做对比分析
   - 生成 **同类药物排名表**（按有效率、副作用严重度、用户满意度排序）

4. **时序分析**
   - 利用 `date` 字段分析药物评价随时间的变化趋势
   - 发现评价突变（可能对应市场事件、配方变更等）

**示例实现**:
```python
# 从评论中提取情感和关键词
from transformers import pipeline
sentiment_analyzer = pipeline("sentiment-analysis")

# 按药物+疾病分组计算有效率
effectiveness = df.groupby(['drugName', 'condition']).agg(
    positive_rate=('sentiment', lambda x: (x == 'POSITIVE').mean()),
    avg_rating=('rating', 'mean')
)
```

## 二、完全未利用的数据集

### DS5: disease-symptom-description-dataset/dataset（疾病→症状映射）

**数据内容**: 4,920 条记录，将 disease_keys.json 中的疾病映射到最多 17 个症状。

**推荐利用方式**:

1. **构建"症状→药物"推理链**
   ```
   用户输入症状 → DS5 匹配疾病 → enhanced_drug_table 推荐药物
   ```
   - 这是完整推荐链路中 **关键的中间环节**
   - 可结合 DS8 的症状权重做加权匹配

2. **构建症状-疾病概率矩阵**
   - 统计每种症状出现在各疾病中的频率
   - 用贝叶斯或决策树做 `P(disease | symptoms)` 概率推断

3. **扩充增强表**
   - 为每种药物添加 `treatable_symptoms` 字段
   - 通过 `drug → disease → symptoms` 的链路反推

**示例实现**:
```python
# 构建疾病-症状矩阵
disease_symptoms = {}
for _, row in ds5.iterrows():
    disease = row['Disease']
    symptoms = [row[f'Symptom_{i}'] for i in range(1, 18) if row[f'Symptom_{i}'] != 'NaN']
    disease_symptoms[disease] = symptoms

# 用户输入症状 → 匹配疾病
def match_disease(user_symptoms, disease_symptoms_dict):
    scores = {}
    for disease, symptoms in disease_symptoms_dict.items():
        score = len(set(user_symptoms) & set(symptoms))
        scores[disease] = score
    return sorted(scores.items(), key=lambda x: -x[1])
```

### DS7: symptom_precaution（疾病预防措施）

**数据内容**: 43 条记录，每种疾病对应 4 条预防/处理建议。

**推荐利用方式**:

1. **药物推荐附加信息**
   - 在推荐药物时，同时展示该疾病的预防措施
   - 如推荐高血压药物时附加："meditation, salt baths, reduce stress, get proper sleep"

2. **构建综合治疗方案**
   - 药物推荐 + 生活方式建议 = 完整的治疗方案输出
   - 字段建议: `precautions`（添加到增强表或作为独立查询表）

**集成方式**:
```python
# 在推荐结果中附加预防措施
def get_recommendation(disease_key):
    drugs = enhanced_table[enhanced_table['matched_disease_keys'].str.contains(disease_key)]
    precautions = ds7[ds7['Disease'] == disease_key][['Precaution_1', 'Precaution_2', 'Precaution_3', 'Precaution_4']]
    return {'drugs': drugs, 'precautions': precautions}
```

### DS8: Symptom-severity（症状严重度权重）

**数据内容**: 135 种症状的严重度权重（1-7分）。

**推荐利用方式**:

1. **加权症状匹配**
   - 用户报告多个症状时，高权重症状应对推荐结果有更大影响
   - 如 `high_fever (weight=7)` 应比 `lethargy (weight=2)` 有更强的推荐信号

2. **紧急程度评估**
   - 根据用户症状的总权重分判断紧急程度
   - 高紧急度时优先推荐处方药（Rx），低紧急度推荐 OTC

3. **构建加权疾病匹配**
   ```python
   def weighted_disease_match(user_symptoms, symptom_weights, disease_symptoms):
       scores = {}
       for disease, symptoms in disease_symptoms.items():
           matched = set(user_symptoms) & set(symptoms)
           score = sum(symptom_weights.get(s, 1) for s in matched)
           scores[disease] = score
       return sorted(scores.items(), key=lambda x: -x[1])
   ```

### DS9: Disease_symptom_and_patient_profile（患者画像）

**数据内容**: 350 条记录，包含年龄、性别、血压、胆固醇水平和疾病结局。

**推荐利用方式**:

1. **个性化药物推荐**
   - 根据用户画像（年龄、性别、血压、胆固醇）调整推荐
   - 如：高血压患者避免推荐含升压成分的药物
   - 如：妊娠期女性优先推荐 pregnancy_category=A/B 的药物

2. **风险评估模型**
   - 训练分类模型预测 `Outcome Variable`
   - 输入: 症状 + 患者画像 → 输出: 疾病确诊概率

3. **人群统计分析**
   - 分析不同年龄段/性别的疾病分布
   - 为特定人群定制推荐策略

**示例**:
```python
# 基于患者画像的个性化推荐
def personalized_recommend(disease_key, age, gender, blood_pressure, enhanced_table):
    candidates = enhanced_table[enhanced_table['matched_disease_keys'].str.contains(disease_key)]

    # 高血压患者过滤
    if blood_pressure == 'High':
        candidates = candidates[~candidates['side_effects'].str.contains('hypertension|blood pressure', case=False, na=False)]

    # 按年龄调整（老年患者优先低副作用）
    if age > 65:
        candidates = candidates.sort_values('avg_rating', ascending=False)

    return candidates
```

### DS10: Symptom2Disease（自然语言症状→疾病）

**数据内容**: 1,200 条记录，自然语言症状描述文本 → 疾病标签。

**推荐利用方式**:

1. **训练症状理解 NLP 模型**
   - 微调 BERT/GPT 做文本分类: 用户自然语言描述 → 疾病标签
   - 作为推荐链路的 **输入层**

2. **增强对话系统**
   - 用户用自然语言描述症状（如 "I have a skin rash on my arms"）
   - 模型识别为 Psoriasis → 查 enhanced_drug_table → 推荐药物

3. **数据增强**
   - 用作训练集，结合 DS5 的结构化症状数据做数据增强
   - 可用 LLM 生成更多类似的训练样本

**示例**:
```python
from transformers import AutoTokenizer, AutoModelForSequenceClassification

# 微调分类模型
model = AutoModelForSequenceClassification.from_pretrained('bert-base-uncased', num_labels=42)
# 训练数据: DS10 的 text → label
# 推理: 用户输入 → 疾病标签 → 药物推荐
```

### DS11: first-aid-QA（急救问答）

**数据内容**: 急救场景的问答对。

**推荐利用方式**:

1. **紧急场景处理**
   - 当用户描述紧急症状（如 heart_attack、paralysis）时
   - 优先展示急救问答内容，再推荐药物

2. **对话系统知识库**
   - 作为 RAG（检索增强生成）的知识库
   - 对话系统可以检索相关急救知识回答用户

```
                    │ (自然语言描述) │
                    └──────┬───────┘

            DS10 →  │  症状理解NLP  │

                    └──────┬───────┘
                           │
              ┌────────────▼────────────┐
      DS5 →   │    症状→疾病匹配引擎     │  ← DS8 (症状权重)
              │  (结构化+加权匹配)       │
              └────────────┬────────────┘
                           │
              ┌────────────▼────────────┐
              │   enhanced_drug_table    │  ← DS1+DS2+DS3+DS4+DS6
              │   (疾病→药物推荐)        │
              └────────────┬────────────┘
                           │
              ┌────────────▼────────────┐
      DS9 →   │    个性化排序 & 过滤     │  ← DS4 (情感分析)
              │  (患者画像+用户评价)      │
              └────────────┬────────────┘
                           │
              ┌────────────▼────────────┐
      DS7 →   │    综合治疗方案输出      │  ← DS11 (急救QA)

              └────────────┘────────────┘
```

> 以上每个环节均可独立开发和集成，建议按从上到下的顺序逐步实现。
